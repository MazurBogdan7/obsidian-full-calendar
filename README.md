# Full Calendar Enhanced

Расширенная версия плагина [obsidian-full-calendar](https://github.com/davish/obsidian-full-calendar) с четырьмя дополнительными функциями.

---

## Установка

### 1. Сборка

```bash
# Установить зависимости (один раз)
npm install

# Собрать плагин
npm run build
```

После сборки появится папка `dist/` с тремя файлами:
```
dist/
├── main.js
├── styles.css
└── manifest.json
```

### 2. Копирование в Obsidian

Скопируй содержимое папки `dist/` в:

```
{твоё хранилище}/.obsidian/plugins/obsidian-full-calendar/
```

> Если плагин уже был установлен — замени `main.js` и `styles.css`. Файл `data.json` (настройки календаря) не трогай.

### 3. Активация

В Obsidian: **Настройки → Сторонние плагины → Full Calendar → включить**

После загрузки появится уведомление «Расширенный календарь активен». Расширенные функции включаются автоматически.

---

## Разработка (watch-режим)

```bash
npm run dev
```

Патч пересобирается автоматически при каждом изменении файлов в `src/`. После пересборки перезагрузи плагин в Obsidian (`Ctrl+P → Reload app without saving`).

## Новые функции

### 🔗 Привязка заметок и папок к событиям

При создании или редактировании события в форме появляется секция **«Связанные заметки/папки»**.

- Нажми **«+ Добавить заметку / папку»** и выбери нужный файл или папку через fuzzy-поиск
- Можно добавить любое количество ссылок
- При **наведении** на событие в календаре всплывает список привязанных заметок — клик открывает заметку
- На событии в календаре отображается значок `🔗N`

**Как хранится** — в frontmatter файла события:
```yaml
linkedNotes:
  - "Проекты/Мой проект.md"
  - "Архив/"
```

**Настройки:** Настройки плагина → Привязка заметок

---

### 🔔 Уведомления со звуком

Obsidian уведомляет о приближающихся и завершающихся событиях.

**Глобальные настройки** (в настройках плагина):
- Включить/выключить уведомления
- За сколько минут предупреждать (0–30 мин, слайдер)
- Уведомлять об окончании событий
- **«▶ Запустить тест»** — принудительно проверить все события прямо сейчас (удобно для отладки)
- Звук начала — путь к файлу `.mp3`/`.wav`/`.ogg` в хранилище + кнопка «Тест»
- Звук конца — аналогично

> Если звук не указан — воспроизводится короткий бип через Web Audio API.

**Настройки для каждого события** (в форме редактирования):
- Включить/выключить уведомление для этого события
- Переопределить звук начала и/или конца

**Как хранится:**
```yaml
notification:
  enabled: true
  startSound: "audio/start.mp3"   # null = глобальный звук
  endSound: null
```

**Отладка:** открой DevTools (`Ctrl+Shift+I`) → Console → ищи строки `[FCEnhanced]`. Они показывают сколько файлов найдено, время до события и срабатывание.

### ⏱ Жёсткий трекинг выполнения

Фиксирует реальное время выполнения события.

**Как работает:**
1. Открой событие в редакторе — появится секция **«⏱ Трекинг выполнения»**
2. Нажми **▶ Приступил к задаче** когда начнёшь — сохраняется реальное время старта
3. Нажми **⏹ Завершил задачу** когда закончишь — сохраняется реальное время конца, запись добавляется в дашборд

> Кнопки **▶/⏹ видны всегда** — можно нажать в любое время, не только в момент наступления события.

**Визуальное отображение в календаре:**
- Опоздал со стартом → верхняя часть события закрашена приглушённым цветом + показано реальное время старта
- Превысил плановое время → событие расширяется вниз + показано реальное время конца

**Если трекинг выключен** для события — оно автоматически добавляется в дашборд после окончания планового времени.

**Как хранится:**
```yaml
tracking:
  enabled: true
  startedAt: "2025-04-03T09:14:22.000Z"
  endedAt: "2025-04-03T10:45:00.000Z"
```

**Настройки:**
- Включать трекинг по умолчанию для новых событий
- Показывать/скрывать кнопки трекинга

### 📊 Дашборд

Отдельная вкладка с аналитикой по всем событиям периода.

**Открыть:** `Ctrl+P → Открыть дашборд календаря` или кнопка в настройках плагина.

**Периоды:** Неделя / Месяц / Год

**Разделы дашборда:**

| Блок | Описание |
|------|----------|
| Сводные карточки | Всего событий, запланировано, выполнено, опоздания, не отслежено |
| Запланировано vs Факт | Canvas-гистограмма по дням/месяцам: светлый = план, тёмный = факт |
| Динамика по активности | Гистограмма для конкретного события/активности; доступны только повторяющиеся события или с одинаковым названием (≥2 раза) |
| Время по активностям | Таблица с поиском и фильтрами, 5 строк + прокрутка |
| Пунктуальность | Таблица опозданий старта/конца |
| Запланированные события | Все события периода с временем и статусом трекинга |
| Все события периода | Полная таблица с плановым и реальным временем |

Все таблицы: поиск по названию + фильтры по столбцам + прокрутка колёсиком (5 строк видно сразу).

**Экспорт:** кнопка «📤 JSON» — сохраняет данные в папку дашборда.

**Настройки:**
- Папка для сохранения данных (по умолчанию `_calendar_dashboard/`)

## События, переходящие через полночь

Для событий типа «Сон» (21:30 → 05:30) плагин автоматически корректирует дату окончания:

- В форме создания/редактирования рядом с временем конца появляется поле **«до [дата]»** — можно вручную выбрать дату окончания
- Если дата окончания не изменена, но `endTime < startTime` — плагин автоматически ставит следующий день
- Гистограммы и дашборд корректно вычисляют продолжительность таких событий

**Пример frontmatter:**
```yaml
date: "2025-04-05"
startTime: "21:30"
endTime: "05:30"
endDate: "2025-04-06"   # Добавляется автоматически или вручную
```

## Frontmatter события — полный формат

```yaml
---
title: "Название события"
date: "2025-04-03"
startTime: "09:00"
endTime: "10:30"
endDate: "2025-04-03"   # Только для событий через полночь
allDay: false
color: "#3788d8"

# Привязанные заметки (функция 1)
linkedNotes:
  - "Проекты/Проект А.md"
  - "Материалы/"

# Уведомления (функция 2)
notification:
  enabled: true
  startSound: "audio/start.mp3"   # null = использовать глобальный
  endSound: null

# Трекинг (функция 3)
tracking:
  enabled: true
  startedAt: "2025-04-03T09:14:22.000Z"
  endedAt: "2025-04-03T10:45:00.000Z"
---
```

## Технические детали (для разработчиков)

### Как работает патч

Сборка объединяет `main.original.js` (оригинальный плагин + 5 патчей прямо в нём) с скомпилированным TypeScript-кодом из `src/`.

**Патчи в main.original.js:**

| Место | Патч | Зачем |
|-------|------|-------|
| `ReactModal.onOpen` | Вызывает `window.__fcOnModalRendered(contentEl, eventId)` после рендера | Позволяет инжектировать UI в модал |
| `launchEditModal` | Сохраняет `eventId` в `__fcEditModal.__fcEventId` | Передаёт ID события в хук |
| `EventStore.add` | Idempotent: повторный `add` с тем же ID обновляет, а не бросает ошибку | Исправляет «already exists» при создании событий |
| `FullNoteCalendar.createEvent` | `try/catch` вокруг `vault.createFolder` | Папка может уже существовать |
| `toEventInput` | Авто-сдвиг `endDate` на следующий день при `endTime < startTime` | Корректное отображение кросс-полуночных событий |
| `launchCreateModal/launchEditModal` submit | Авто-вычисление `endDate = date+1` при `endTime ≤ startTime` | Корректное сохранение кросс-полуночных событий |
| Форма `EditEvent` | Добавлено поле `[дата окончания]` после времени конца | Ручное управление датой конца |

### event.id vs путь к файлу

В FullCalendar `event.id` — это счётчик EventStore (`"0"`, `"1"`, …), **не путь к файлу**.

Для получения пути:
```ts
plugin.cache.getInfoForEditableEvent(eventId)?.location?.path
```

### Директории в настройках

Если в настройках указана директория `"/"` (корень хранилища), все фильтры файлов обрабатывают это как «все `.md` файлы».

### Блокировка файлов

`withFileLock(filePath, fn)` — per-file sequential write lock, предотвращает одновременные `processFrontMatter` на одном файле.
